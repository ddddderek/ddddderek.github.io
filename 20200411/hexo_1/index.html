    从零开始搭建个人博客（一）
    2020/04/11 Usage Hexo Github

    相信大多数开发者都曾想拥有一个个人博客，记录自己生活所见、工作感悟、人生感想，当然也可能是为了炫技和装X。但是服务器地选配，页面、业务流程的设计和开发等带来的成本，使大家望而却步。本文将使用 Hexo和 Github 带你实现“零成本”的个人博客，让你更多的专注在写作本身。

    什么是Hexo

    Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

    来源：Hexo文档

    现在我们已经可以创建、编辑文章，并能在本地服务上预览我们的博客了。但是写博客不就是为了与大家进行技术交流嘛（装X），况且我们代码保存在本地，万一我们的硬盘坏了岂不是多年心血付之一炬，由此便引出了接下来的内容——Github（江湖人称大型男性交友网站）。

    创建仓库

    如果你已经认真阅读了 hexo 的官方文档，那么你一定会看到如下建议：

    一个好的实践是将站点目录和 Pages 分别存放在两个不同的 Git 仓库中，可以有效避免相互覆盖。
    Hexo 在部署你的站点生成的文件时并不会更新你的站点目录，因此你应该手动提交并推送你的写作分支。

    来源：Hexo文档

    创建 pages 仓库
